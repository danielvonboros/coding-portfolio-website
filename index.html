Case study for filmOpedia React Application

THE CHALLENGE

To create a portfolio project from scratch, developing both backend and frontend and connecting the server-side to the client side. Develop the application using the MERN stack. filmOpedia is designed as an online movie library that should provide data about movies, genres and directors as well as personal information about the user. Users should be able to create a user account with a username, email and birthday as well as a list of favourite movies.

PURPOSE & CONTEXT

filmOpedia was created as a part of the Full-Stack-Development course at CareerFoundry. Both server-side and client-side had to be created, to showcase the skills in handling databases and creating a React Web Application from scratch. My role was to act as a full-stack developer on the project, creating back-end, front-end and style the application with an attractive user interface.

THE PROCESS

Set up backend-logic for creating, reading, updating and deleting data stored on a server and connect these processes to the frontend. Then create a simple to use and nice looking application that displays the data from collections that are being stored on the server.

TOOLS USED

The project was created using the MERN (MongoDB, Express, React, Node) stack.

BACKEND DEVELOPMENT

The backend is designed to contain two collections: one for the users and another one for the movies. Both collections contain data in JSON format. The backend was designed to allow create a user and read from users collection requests without authorisation (to create users and allow login requests). To prevent the application from asking for username and password at any request a user makes, password hashing and a JWT-token were added to the page, that is valid for a week, so users only have to login once a week and will be automatically authenticated by the token in between. The login request itself uses HTTP authentication. To test the API I used Postman to test the endpoints and all requests.

FRONTEND DEVELOPMENT

After completing the backend for the application, the frontend for the application is created and designed. It was my first approach to React, it is bundled with Parcel and styled with React Bootstrap. To be able to access different routes directly from the web address input, React Router is added to the webpage to provide direct access to certain components, as well as a direct URL for each object in the collection. The application itself provides different views, the main view provides information about all movies, from here several different views can be accessed: a profile view (to display, edit or delete user information and information about favourite movies), a single movie view (to display a description of one selected movie), a genre view (to display information about a movie genre), a director view (to provide information on a director) and a registration view and login view, where users can register and registered users are able to sign-in.





CHALLENGES

After developing the backend of the application was effective and a lot of fun, during the development of the frontend I faced most challenges I ever faced in developing a portfolio project out of the CareerFoundry bootcamp. It made me more than once doubt about my abilities in coding. With the help of my tutor I was able to solve the challenge in the end. But after seeing my tutor having as well problems at one point or another, my scepsis towards my coding skills decreased. In the end it turned out to be a great training for me to do the project for a second time, and I was able to wrap my head around the tasks a lot faster.

DURATION

Developing this application (both front-end and back-end) took me around 6 weeks in the end. I’ve been stuck at several points in the development of the application. When the task was finally ready for submission, I had learned a lot and learned a lot when facing probably the most difficulties through the entire process.
